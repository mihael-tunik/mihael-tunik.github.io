Count inversions with Fenwick trees

Counting inversions in O(n log n) is infamous algorithmic question with divide and conquer solution. However, did you know that there exists a family of algorithms to solve the it with same asymptotics? Let’s find out!

Inversions

Let a be our array and i, j indexes.

By definition number of inversions is number of pairs (i, j) for which following predicate is true: a[i] > a[j] and i < j.

Simple quadratic algorithm can be easily constructed:

def inversions_base(a):
    n, cnt = len(a), 0
    for i in range(0, n):
        for j in range(0, n):
            if a[i] > a[j] and i < j:
                cnt += 1
    return cnt

How to progress from it? First we need to reformulate counting task in terms of flag array.

def inversions_flag_array(a):
    n, cnt, M = len(a), 0, max(a) 
    T = [0] * (M + 1)
    for i in range(0, n):
        T[a[i]] += 1 # note, exactly +1
        for j in range(a[i] + 1, M+1):
            cnt += T[j]
    return cnt

What is going on here? We iterate over array elements one by one in order and use auxiliary array T of flags. Update T[k] += 1 if we meet k = a[i] in array (for distinct arrays T[k]=1 is enough). Then we can count number of occurrences for elements which are larger than a[i] but located before easily by watching T. Since T is ordinary array we can simply iterate from a[i]+1 to max(a). That results in quadratic solution.

Certainly we can do better, note that inner part of main cycle is a sum query. And as we know (don’t we?) that such queries on array interval can be made more efficient with help of data structures. Fenwick tree here is certainly a way to go. Let’s do it, start from class for our tree.

Fenwick tree

First discuss Fenwick tree a little bit. …

class FenwickTree:
    def __init__(self, n):
        self.t = [0] * n # long empty list 
        self.N = n
        
    def modify(self, i, d):
        while i < self.N:
            self.t[i] += d
            i = i | (i + 1)
    
    def prefix_sum(self, i):
        result = 0
        while i >= 0:
            result += self.t[i]
            i = (i & (i + 1)) - 1
        return result

    def query(self, i, j):
        return self.prefix_sum(j) - self.prefix_sum(i - 1)

Back to flag arrays

Now flag array code snippet transforms into:

# O(n log n)
def inversions_fenwick(a):
    n, cnt, M = len(a), 0, 20
    T = FenwickTree(M+1)        
    for i in range(0, n):
        T.modify(a[i], 1)
        cnt += T.query(a[i] + 1, M)

    return cnt

And that’s it. This simple idea has caught my attention recently, because it is not well known as classic mergesort-like algorithm.
