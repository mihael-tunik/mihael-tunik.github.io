Integer labeling

Jun 19, 2024 · 488 words · 3 minute read

Everybody knows about marching squares and cubes.

These typically used for contour line/surface computation. In such cases function f acts from R^2 -> R and zero set problem consists of one equation. But what if f is vector-function and acts, for example, from R^3 -> R^2? Surprisingly, there’s nice family of algorithms to handle this case, called labeling methods.

Three years ago in my project I implemented one such algorithm, called integer labeling. Let’s see what is going on in this code and where magic happens.

int label_fast(d_pair f_val){ 
    int map[] = {1, 2, 2, 4};
    int r = ((f_val.data[1] >= 0.0) << 1) | (f_val.data[0] >= 0.0);        
    return map[r];
}
void label_fast_vec(double *slice_data, char *cache, int size){
    int NN = size*size;
    for(int l = 0; l < NN; ++l){
        d_pair f_val;       
        f_val.data[0] = slice_data[l], f_val.data[1] = slice_data[NN+l]; 
        cache[l] = label_fast(f_val);
    }
}
extern "C" int integer_labeling_3d_vec(grid_t GridDesc, void (*ptr)(int, double, double *), 
                                   double *result, int *size){   
    /* init some data */
    label_fast_vec(f_wrapped_vec(N, z[0], ptr, slice_data_ptr), cache_prev, N);
    for(int k = 0; k < Nz-1; ++k){  
        label_fast_vec(f_wrapped_vec(N, z[k+1], ptr, slice_data_ptr), cache_next, N);
        for(int l = 0; l < NN; )
            if((l / N) != N-1 && (l % N) != N-1){
                char label_flags = (cache_prev[l] | cache_prev[l + N] | cache_next[l] | cache_next[l + N]);
                ++l;
                label_flags |= (cache_prev[l] | cache_prev[l + N] | cache_next[l] | cache_next[l + N]);  
                if(COMPLETELY_LABELED(label_flags)){
                    result[3*cnt    ] = x[(l - 1) / N];
                    result[3*cnt + 1] = y[(l - 1) % N];
                    result[3*cnt + 2] = z[k];
                    ++cnt;
                }
            }else
                 ++l;
        tmp        = cache_prev;
        cache_prev = cache_next;
        cache_next = tmp;
    }
    /* clean up some data */       
    return 0;    
}

So it’s labeling algorithm and that means that for every grid vertice we compute some label values and then according to them decide what elements will present the zero set of our function. Suppose we have simplicial space subdivision.

According to Introduction to Numerical Continuation Methods by Eugene L. Allgower and Kurt Georg appropriate labeling can be defined as simple as number of positive coordinates. Then chosen simplex is called completely labeled if set of its vertices contains every possible labeling. This result deeply connected with Sperner’s Lemma and that is quite interesting result.

Let’s return to our code. Labeling data of cube layer stored in cache_prev, cache_next variables. On every iteration we load two consecutive function data slices. Cube labeling set is encoded by bits in label_flags variable. Basically, here we sum up some integer values for every cube of our grid in some speed-optimized manner.

   label_flags |= (cache_prev[l] | cache_prev[l + N] | cache_next[l] | cache_next[l + N]);

      /* cube is found */
   }

Macros COMPLETELY_LABELED(…) is defined simply as

#define COMPLETELY_LABELED(x) ((x) == 0x7)

and that’s it.
